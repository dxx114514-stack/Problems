玻璃森林题解

关键观察

若知道一段序列的：

  • 最左端颜色；
  • 最右端颜色；
  • 内部连续颜色段数量；

就可以合并两段相邻序列。若左段右端颜色等于右段左端颜色，合并后段数需要减一。

因此可以在线段树中维护这三个信息，再用树链剖分把树上路径拆成若干条重链区间。

算法

对树做重链剖分，将每个点映射到 DFS 序位置。线段树的每个节点维护：

  • lc：区间按 DFS 序从左到右的第一个颜色；
  • rc：区间按 DFS 序从左到右的最后一个颜色；
  • cnt：区间颜色段数量。

点修改就是线段树单点修改。

路径询问时，路径方向很重要：

  • 从 \(u\) 往上跳的重链区间，在 DFS 序里是从链顶到 \(u\)，但路径顺序是从 \(u\) 到链顶，所以要反转左右端颜色；
  • 从 \(v\) 往上跳得到的区间，最终应接在答案右侧，因此需要按从链顶到 \(v\) 的顺序拼到右半答案前面。

最后同一条重链上的中间区间也按实际路径方向处理，再合并左右两部分。

正确性说明

线段树节点信息足以描述一段颜色序列与相邻序列合并时会产生的影响。合并公式只在两段交界处可能改变段数，因此维护的信息完备。

树链剖分将任意树上路径拆成若干个不交的链上连续区间。查询过程中分别维护从原始 \(u\) 出发的左半路径和通向原始 \(v\) 的右半路径，并在每段加入前调整方向，所以合并顺序与真实路径颜色序列一致。最终得到的段数就是题目所求。

复杂度

预处理 \(O(n)\)，每次修改 \(O(\log n)\)，每次询问拆成 \(O(\log n)\) 条链，每条链做一次线段树查询，因此复杂度为 \(O(\log^2 n)\)。

空间复杂度 \(O(n)\)。

易错点

  • 路径查询不是无向集合查询，颜色段数依赖方向。
  • 合并两段时，只有交界颜色相同才减一。
  • 修改的是点颜色，不是边颜色。
