# T762052 基因编辑的碱基配对

## 题目背景

kimi

## 题目描述

科学家获得了一条 DNA 序列，由 $n$ 个碱基组成，每个碱基是 `A`、`T`、`C`、`G` 之一。

基因编辑仪器可以执行一种特殊的**排序转录**操作：给定区间 $[l,r]$，将该区间内的所有碱基按照 `A` < `T` < `C` < `G` 的字典序排序后，按原顺序重新填充回该区间。

为了验证编辑结果，科学家需要随时查询某段区间 $[l,r]$ 的**基因指纹**（即排序后该区间的哈希值）。

哈希规则如下：设排序后的碱基序列为 $b_1,b_2,\dots,b_L$（$L=r-l+1$），定义其哈希值为
$$H = \sum_{i=1}^{L} val(b_i) \cdot P^{i-1} \pmod{998244353}$$
其中 $val(`A`)=1, val(`T`)=2, val(`C`)=3, val(`G`)=4$，$P$ 是给定的正整数。

操作有两种：

- `1 l r`：对区间 $[l,r]$ 进行排序转录。
- `2 l r`：查询区间 $[l,r]$ 的基因指纹 $H$

## 输入格式

第一行三个整数 $n,m,P$。

第二行一个长度为 $n$ 的字符串，表示初始 DNA 序列。

接下来 $m$ 行，每行一个操作，格式如题所述。

## 输出格式

对于每个操作 `2`，输出一行一个整数，表示基因指纹 $H$。

## 输入输出样例 #1

### 输入 #1

```
4 3 10
TCAG
2 1 4
1 2 3
2 2 4
```

### 输出 #1

```
4321
431
```

## 说明/提示

- 初始序列 `TCAG`，排序后 `ATCG`（按 `A<T<C<G`）。
  - $H = 1\cdot 10^0 + 2\cdot 10^1 + 3\cdot 10^2 + 4\cdot 10^3 = 1 + 20 + 300 + 4000 = 4321$。
- 操作 `1 2 3`：区间 `CA` 排序后 `AC`，序列变为 `T A C G`。
- 查询 `2 2 4`：区间 `ACG`，排序后 `ACG`（已有序）。
  - $H = 1\cdot 10^0 + 3\cdot 10^1 + 4\cdot 10^2 = 1 + 30 + 400 = 431$。

## 数据范围

- $1 \le n,m \le 2\times 10^5$
- $2 \le P < 998244353$
- 字符串只包含 `A`、`T`、`C`、`G`
