## 题解：神经网络的突触重构（SynapseRebuild）

### 核心观察

本题要求支持区间加和区间最小值查询，这是一个经典的线段树问题。

定义突触强度数组 $w_1, w_2, \dots, w_{n-1}$，其中 $w_i$ 表示神经元 $i$ 与 $i+1$ 之间的连接强度。

对于查询操作 `2 l r`，需要计算 $\min_{i=l}^{r-1} w_i$，即区间 $[l, r-1]$ 的最小值。

对于修改操作 `1 l r x`，需要将区间 $[l, r-1]$ 内所有元素增加 $x$。

### 算法

使用**线段树**维护区间最小值，支持懒标记（lazy propagation）来实现区间加法。

线段树的每个节点维护：
- `min_val`：该区间内的最小值
- `lazy`：懒标记，表示该区间需要整体增加的值

**关键实现细节**：
- 使用 `vector` 动态分配线段树空间，避免全局大数组导致栈溢出
- `tree.resize(size * 4)` 在运行时动态分配内存
- 懒标记下传时同步更新子节点的 `min_val` 和 `lazy`

操作：
- **区间加**：在线段树上对区间 $[l, r-1]$ 执行加法操作，时间复杂度 $O(\log n)$。
- **区间最小值查询**：在线段树上查询区间 $[l, r-1]$ 的最小值，时间复杂度 $O(\log n)$。

### 复杂度

- 时间：$O(n + m\log n)$，每次操作 $O(\log n)$。
- 空间：$O(n)$，线段树使用 vector 动态分配。

### 注意事项

- 查询区间 $[l,r]$ 对应的是突触区间 $[l, r-1]$，需要注意边界。
- 使用 vector 而非全局数组避免 Windows 栈溢出问题（默认栈限制 1-8MB）。
- 懒标记下传时要正确更新子节点的 `min_val` 和 `lazy`。
