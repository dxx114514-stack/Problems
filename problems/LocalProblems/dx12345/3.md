# T709789 Steve In Minecraft

## 题目描述

玩家有 4 个属性：**饥饿值**、**饱和度**、**血量**、**经验**

### 属性规则

- **饥饿值**: 范围 0~20，初始 20
- **饱和度**: 范围 0~当前饥饿值，初始 5
- **血量**: 初始 20
- **经验等级**: 初始 0

### 消耗度机制

消耗度初始为 0，每累计 4 点消耗度会扣除 1 点饱和度或饥饿值:
- 若饱和度 > 0: 优先扣除饱和度 (不足 1 不扣饥饿值)
- 若饱和度 = 0: 扣除 1 点饥饿值

### 回血机制

- 每秒可用 6 消耗度回复 1 血量 (饱食度满后)
- 每 0.5 秒可在饱和度用完后，饥饿值≥18 时用 6 消耗度回复 1 血

### 死亡重置

死亡时：血量=20，饱食度=20，饱和度=0，经验等级=0，死亡次数 +1

### 进食机制

当饥饿值 < 20 时可以进食，进食后立即恢复对应的饥饿值和饱和度：
- 恢复的饥饿值不能超过 20
- 恢复的饱和度不能超过恢复后的饥饿值
- 若饥饿值已满 (20)，则无法进食，该操作无效

## 输入格式

一系列操作:
- `Break [block]` - 挖掘方块
- `Swim [meter]` - 游泳
- `Jump` - 跳跃
- `Sprint [meter]` - 冲刺
- `Drop [meter]` - 下落
- `Kill [mob]` - 攻击生物
- `no_oxygen [sec]` - 窒息
- `lava [sec]` - 岩浆
- `on_fire [sec]` - 着火
- `Atted_by [mob]` - 被生物攻击
- `Eat [food]` - 进食
- `Query` - 查询状态

## 输出格式

每次 Query 输出一行:
```
死亡次数 血量 饱食度 饱和度 经验等级
```

## 样例

### 输入
```
Query
Break amethyst_cluster
Break amethyst_cluster
Query
Drop 24
Query
```

### 输出
```
0 20 20 20 0
0 20 20 20 1
1 20 20 20 0
```

## 数据范围

操作次数 ≤ 10^4
