# T748841 预备——又是旅行算路费

## 题目背景

> 此题为下一次欢乐赛可选题

ZSC在SYC 的土地上旅行，他需要经过许许多多的高速公路，这些公路的路费令他头疼。庆幸的是，他拥有许多优惠券，可以在任何道路上使用，使用完成之后会使这一段的路费清零。他现在做旅行计划，请你帮他算一算到达终点需要多少钱。现在Jack觉得这题太简单了，想难倒小朋友们。于是，他将一个枢纽的描述改成了一个复杂的串。同时，他也改变了高速公路。不仅如此，ZSC的手机导航也坏掉了。请你帮帮他，让他到达景点。

## 题目描述

给出一张高速公路路网图**G**。\
现在$ZSC$想从`home`到达`attraction`。优惠券的数量有限，~~不然*SYC* 就没收入了~~。\
但是，如果高速公路的设计者很坏，$ZSC$到不了景点，请你输出一个令人失望的`-1`。\
高速公路的设计者为了描述枢纽，设计了这样的一个结构：\
形如：
`!#!"name"="<name>" "id"=<id> "type"="<type>"!#!`
- `<name>`表示枢纽名称，为一个含空格字符串
- `<id>`为一个0x开头的**小写**十六进制整数，其十进制形式**应小于等于**$2^{64}$
- `<type>`为一个字符串，应为:
  - `open` 表示枢纽开放
  - `close`表示枢纽不开放，禁止驶入
  - 若一个节点类型为 `limit`，则所有与该节点**直接相连**的道路，均**不允许**使用优惠券。换句话说：一条道路只要**任意一端**是 limit 节点，这条道路就**永远不能**使用优惠券。

高速公路的设计者十分粗心，他不小心把文件损坏了，添加了一大堆不合法的枢纽。请你判断，哪些枢纽描述合法。对于不合法的枢纽，你不能驶入，以免发生事故。

高速公路的描述如下：\
`!@!"id"="<id>" "direction"="<dir>" "u"=<u> "v"=<v> "w"=<w> !@!`
- `<id>`表示高速公路名称，为一个含空格字符串
- `<direction>`为一个字符串，仅有可能为:
  - `all` 表示高速公路可双向通行
  - `close`表示高速公路不开放，禁止驶入
  - `unidirectional`表示枢纽可以行驶，但是仅允许从`<u>`走向`<v>`
- `<u>`为一个以`0x`开头**大写**十六进制整数，表示这条高速公路的起始点枢纽id，不一定为出现过的枢纽，如果没出现过，请忽略这条高速公路。
- `<v>`为一个以`0x`开头**大写**十六进制整数，表示这条高速公路的终止点枢纽id，不一定为出现过的枢纽，如果没出现过，请忽略这条高速公路。
- `<w>`为一个以0x开头的**小写**十六进制整数，表示走这条高速公路的花费。其十进制形式**小于等于**$2^{32}$。
:::info{open}
当`<u>`或`<v>`指向的枢纽不合法时，这条高速公路不可驶入。
:::

----
接下来，你需要帮助ZSC输出从家到景区的花费及路径。

## 输入格式

第一行输入一个整数 $k$ ,表示有 $k$ 张优惠券。\
接下来 $T$ 行，每行一个字符串，表示枢纽或高速公路。
:::info{open}
请使用较快的输入输出方式。
:::

## 输出格式

第一行输出一个十进制整数：表示最小花费。\
接下来,请你依次输出ZSC会经过的枢纽名称，每行输出一个节点。

## 输入输出样例 #1

### 输入 #1

```
1
!#!"name"="home" "id"=0x1 "type"="open"!#!
!#!"name"="attraction" "id"=0x2 "type"="open"!#!
!#!"name"="middle" "id"=0x3 "type"="limit"!#!
!#!"name"="bad node" "id"=0x4!#!
!@!"id"="road1" "direction"="all" "u"="home" "v"="middle" "w"=0xa !@!
!@!"id"="road2" "direction"="unidirectional" "u"="middle" "v"="attraction" "w"=0x64 !@!
!@!"id"="road3" "direction"="all" "u"="home" "v"="attraction" "w"=0xc8 !@!
!@!"id"="road4" "direction"="close" "u"="home" "v"="bad node" "w"=0x1 !@!
```

### 输出 #1

```
0
home
attraction

```
